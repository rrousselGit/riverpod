[ProviderObserver], uygulamadaki provider yaşam döngüsü olaylarını gözlemlemek için kullanılan bir nesnedir.
Genellikle günlük kaydı (logging), analitik veya hata ayıklama amacıyla kullanılırlar. 

## Kullanım {#usage}

Bir [ProviderObserver] kullanmak için sınıfı genişletmeniz ve gözlemlemek istediğiniz yaşam döngüsü metotlarını geçersiz kılmanız gerekir.
Kullanılabilecek çok sayıda metot vardır. Ayrıntılar için [dokümantasyonunu][ProviderObserver] incelemeniz önerilir.

## Örnek: Logger {#example-logger}

Aşağıdaki gözlemci, uygulamadaki herhangi bir provider'ın tüm durum değişikliklerini günlüğe kaydeder:

<CodeBlock>{trimSnippet(logger)}</CodeBlock>

Artık provider'ımızın değeri her güncellendiğinde logger bunu kaydedecektir:

```
{
  "provider": "Provider<int>",
  "newValue": "1"
}
```

Hata ayıklamayı kolaylaştırmak için provider'larınıza isteğe bağlı olarak bir ad verebilirsiniz:  
```dart
final myProvider = Provider<int>((ref) => 0, name: 'MyProvider');
```

Bu değişiklikle birlikte günlük kaydı şuna dönüşür:
```
{
  "provider": "MyProvider",
  "newValue": "1"
}
```

:::tip
Kod üretimi kullanıldığında provider'lara otomatik olarak bir ad atanır.
:::


:::note NOT
Bir provider'ın durumu yerinde değiştirilirse (genellikle [Ref.notifyListeners] ile birlikte kullanılan List'lerde),
`didUpdateProvider` metodunun `previousValue` ve `newValue` değerlerini aynı değer olarak alması olasıdır.

Bunun nedeni, Dart'ın nesneleri "referans" yoluyla güncellemesidir.
Bunu değiştirmek isterseniz, nesnelerinizi değiştirmeden önce kopyalamanız gerekir.
:::
