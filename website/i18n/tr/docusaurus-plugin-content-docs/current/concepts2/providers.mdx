Provider'lar Riverpod'un merkezî özelliğidir. Riverpod kullanıyorsanız, onu provider'ları için kullanıyorsunuzdur.

## Provider nedir? {#what-is-a-provider}

Provider'lar özünde, üzerine biraz şeker serpiştirilmiş "hafızalanmış (memoized) fonksiyonlardır".  
Bunun anlamı şudur: provider'lar, aynı parametrelerle çağrıldıklarında önbelleğe alınmış bir değer
döndüren fonksiyonlardır.

Provider kullanmanın en yaygın kullanım senaryosu bir ağ isteği yapmaktır.  
Bir API'den kullanıcı çeken bir fonksiyon düşünün:

```dart
Future<User> fetchUser() async {
  final response = await http.get('https://api.example.com/user/123');
  return User.fromJson(response.body);
}
```

Bu fonksiyonun bir sorunu şu: onu widget'ların içinde kullanmaya kalksaydık,
sonucu kendimiz önbelleğe almamız; ardından değeri ihtiyaç duyan tüm widget'lar arasında
paylaşmanın bir yolunu bulmamız gerekirdi.

İşte provider'lar burada devreye giriyor. Provider'lar fonksiyonların etrafındaki sarmalayıcılardır. 
Söz konusu fonksiyonun sonucunu önbelleğe alır ve birden fazla widget'ın
aynı değere erişmesine izin verirler:

<AutoSnippet
  language="dart"
  codegen={`
  // fetchUser fonksiyonumuzun eşdeğeri, ancak sonuç önbelleğe alınır.
  // Bu bir "userProvider" üretecektir. Onu birden çok kez kullanmak
  // aynı değeri döndürecektir.
  @riverpod
  Future<User> user(Ref ref) async {
    final response = await http.get('https://api.example.com/user/123');
    return User.fromJson(response.body);
  }
  `}
  raw={`
  // fetchUser fonksiyonumuzun eşdeğeri, ancak sonuç önbelleğe alınır.
  // userProvider'ı birden çok kez kullanmak aynı değeri döndürecektir.
  final userProvider = FutureProvider<User>((ref) async {
    final response = await http.get('https://api.example.com/user/123');
    return User.fromJson(response.body);
  });
  `}

Temel önbelleklemenin üzerine provider'lar, kendilerini daha güçlü kılmak için
çeşitli özellikler ekler:

- **Yerleşik önbellek geçersizleştirme mekanizmaları**  
  Özellikle [Ref.watch](https://pub.dev/documentation/riverpod/latest/riverpod/Ref/watch.html),
  önbellekleri birbiriyle birleştirmenize ve gerekli olanı otomatik olarak geçersizleştirmenize olanak tanır.
- **<Link documentID="concepts2/auto_dispose" />** <br/>
  Provider'lar artık gerek duyulmadığında kaynakları otomatik olarak serbest bırakabilir.
- **Veri bağlama**  
  Provider'lar [FutureBuilder](https://api.flutter.dev/flutter/widgets/FutureBuilder-class.html)
  veya [StreamBuilder](https://api.flutter.dev/flutter/widgets/StreamBuilder-class.html) ihtiyacını ortadan kaldırır.
- **Otomatik hata yönetimi**  
  Provider'lar hataları otomatik olarak yakalayıp kullanıcı arayüzüne sunabilir.
- **Sahteleme desteği**  
  Daha iyi test edebilmek ve başka amaçlar için tüm provider'lar sahtelenebilir.
  Bkz. <Link documentID="concepts2/overrides" />.
- **<Link documentID="concepts2/offline"/>** <br/>
  Bir provider'ın sonucu diske kalıcı olarak yazılabilir ve uygulama yeniden
  başlatıldığında otomatik olarak geri yüklenebilir.
- **<Link documentID="concepts2/mutations" />** <br/>
  Provider'lar, form gönderimi gibi yan etkiler için kullanıcı arayüzlerinin bir yükleme göstergesi/hata durumu göstermesine yönelik yerleşik bir yol sunar.

Provider'lar 6 çeşitte gelir:

|              | Senkron      | Future                | Stream                 |
| ------------ | ---------------- | --------------------- | ---------------------- |
| Değiştirilemez | [Provider]         | [FutureProvider]        | [StreamProvider]         |
| Değiştirilebilir   | [NotifierProvider] | [AsyncNotifierProvider] | [StreamNotifierProvider] |

Bu ilk bakışta göz korkutucu görünebilir. Adım adım açalım.

**Sync** ile **Future** ile **Stream** karşılaştırması:  
Bu tablonun sütunları, fonksiyonlar için Dart'ın yerleşik türlerini temsil eder.
```dart
int synchronous() => 0;
Future<int> future() async => 0;
Stream<int> stream() => Stream.value(0);
```

**Değiştirilemez ile Değiştirilebilir**:  
Varsayılan olarak provider'lar widget'lar tarafından değiştirilemez. Provider'ların "Notifier"
çeşidi onları dışarıdan değiştirilebilir kılar.  
Bu, özel (private) bir setter'a benzer ("değiştirilemez" provider'lar)
```dart
// _state içeriden değiştirilebilir
// ancak dışarıdan değiştirilemez
var _state = 0;
int get state => _state;
```
Buna karşılık açık (public) bir setter ("değiştirilebilir" provider'lar)
```dart
// Her şey "state"i değiştirebilir
var state = 0;
```

:::info
**Değiştirilemez** ile **değiştirilebilir** ayrımını, ilke olarak sırasıyla `StatelessWidget` ile `StatefulWidget` gibi de düşünebilirsiniz.  

Bu tamamen doğru değildir; çünkü provider'lar widget değildir ve her iki tür de "durum" saklar.
Ancak ilke benzerdir: "Tek nesne, değişmez" ile "İki nesne, değiştirilebilir".
:::


## Provider oluşturmak {#creating-a-provider}

Provider'lar "en üst düzey" bildirimler olarak oluşturulmalıdır.  
Bu, herhangi bir sınıfın veya fonksiyonun dışında bildirilmeleri gerektiği anlamına gelir.

Bir provider oluşturmanın sözdizimi, yukarıdaki tabloya göre onun "değiştirilebilir" mi yoksa
"değiştirilemez" mi olduğuna bağlıdır.

## Provider'ları kullanmak {#using-providers}

Provider'lar, kendi başlarına hiçbir şey yapmamaları bakımından widget'lara benzer.  
Widget'ların kullanıcı arayüzünün bir tarifi olması gibi, provider'lar da durumun
bir tarifidir.  
Şaşırtıcı biçimde bir provider tamamen durumsuzdur ve sözdizimini biraz daha
uzatacak olmasa `const` olarak örneklenebilirdi.

Bir provider'ı kullanmak için ayrı bir nesneye ihtiyacınız vardır: [ProviderContainer]. 
Daha fazla bilgi için <Link documentID="concepts2/containers" /> sayfasına bakın.

Kısacası, bir provider'ı kullanabilmeniz için önce Flutter uygulamalarınızı
bir [ProviderScope] ile sarmalayın:

```dart
void main() {
  runApp(ProviderScope(child: MyApp()));
}
```

Bu yapıldıktan sonra, provider'larınızla etkileşime girmek için bir [Ref] elde etmeniz gerekecek. 
Bunlar hakkında bilgi için <Link documentID="concepts2/refs" /> sayfasına bakın.

Kısaca, bir [Ref] elde etmenin iki yolu vardır:
- Provider'lar doğal olarak bir tanesine erişir.  
  Bu, provider fonksiyonunun ilk parametresi ya da bir Notifier'ın `ref` özelliğidir.
  Bu, provider'ların birbirleriyle iletişim kurmasını sağlar.
- Widget ağacı ise <Link documentID="concepts2/consumers" /> adı verilen özel bir widget türüne ihtiyaç duyar.
  Bu widget'lar size bir [WidgetRef] vererek widget ağacı ile provider ağacı arasındaki boşluğu doldurur.

Örnek olarak, basit bir string döndüren bir `helloWorldProvider` düşünün.
Onu widget'ların içinde şöyle kullanabilirsiniz:

```dart
class Example extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return Consumer(
      builder: (context, ref, _) {
        // Provider'ın değerini elde et
        final helloWorld = ref.watch(helloWorldProvider);

        // Değeri kullanıcı arayüzünde kullan
        return Text(helloWorld);
      },
    );
  }
}
```
