### \[Sync/Async]\[Stateless/Stateful]Provider

- <i><u>Note:</u></i> This is just a way of thinking, not actual types. You don’t need to pick
  a type when using code-generation variant anyway!

- `Sync/Async` is being decided by your return type, either of your function or the [build] method
  in case of using a class (stateful):
   - `Sync` ⇒ T `Async` ⇒ Future<T\> or Stream<T\>

- `Stateless/Stateful` is being decided by whether you need to expose methods to modify the state or not (side-effects):
   - `Stateless` ⇒ use a function. `Stateful` ⇒ use a class with [build] method + side effect methods.


### For non code-generation variant:
You have to manually determine the type of your provider:

- Sync Stateless Provider ⇒ [Provider]
- Sync Stateful Provider ⇒ [NotifierProvider]
- Async Stateless Provider ⇒ [FutureProvider] / [StreamProvider]
- Async Stateful Provider ⇒ [AsyncNotifierProvider] / [StreamNotifierProvider]

These providers are covered in-details at [All Providers](providers/provider).
